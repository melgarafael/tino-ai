Tutorial do Tino

Um curador de novidades de IA que roda 100% na sua máquina, lê o seu vault Obsidian, e só te mostra o que realmente importa pro teu momento — hoje.

Este guia leva do git clone até o primeiro refresh populado com novidades ranqueadas do teu jeito. Menos de 10 minutos no total.

Como o Tino funciona por baixo

Três peças, todas locais:

  1. Claude Code é o backend. Skills, agents e prompts vivem em .claude/. Não tem servidor, não tem API.
  2. Teu vault Obsidian é o banco de dados. O Tino lê o vault inteiro como contexto e escreve de volta numa pasta dedicada Tino/.
  3. dashboard.html é um single-file HTML que lê Tino/novidades/ via File System Access API. Zero build, zero deploy.

Fluxo de dados

setup › scaneia vault inteiro → escreve Tino/_perfil.md refresh › lê perfil + coleta RSS → escreve Tino/novidades/*.md dashboard › lê Tino/novidades/ → renderiza UI thumbs-down › escreve Tino/_ajustes.md → treina o próximo ranking
Importante. O dashboard é apenas visor. Conectar o vault no dashboard não dispara o pipeline — só mostra o que o tino refresh já escreveu lá. Se nunca rodou refresh pro vault, o dashboard fica vazio.

Instalação em 4 passos

1

Clone o repo

git clone https://github.com/melgarafael/tino-ai.git
cd tino-ai
2

Instale as deps do Node

npm install

São duas: fast-xml-parser (RSS) e yaml (config). Sem bundler, sem framework.

3

Instale o Tino globalmente

bash install.sh

O script:

  • Symlinka bin/tino → ~/.local/bin/tino (entra no PATH)
  • Symlinka .claude/commands/tino-*.md → ~/.claude/commands/ (slash commands funcionam em qualquer projeto do Claude Code)
  • Symlinka os 3 agents
  • Salva TINO_HOME em ~/.tino/config.sh

Tudo reversível via uninstall.sh. Nenhum arquivo do teu vault é tocado.

PATH. Se ~/.local/bin não está no teu $PATH, o install avisa. Adicione ao teu shell rc: 
export PATH="$HOME/.local/bin:$PATH"
4

(opcional) Chromium para testes E2E

npx playwright install chromium

Só se quiser rodar npm run test:e2e. Se o plano for só usar, pode pular.

Primeiro uso

Depois do install, você tem duas formas equivalentes de invocar o Tino:

  • Slash commands dentro do Claude Code (funcionam de qualquer pasta): /tino:setup, /tino:refresh, /tino:profile-sync, /tino:deep-dive
  • CLI direto do terminal: tino setup, tino refresh, tino dashboard:open

/tino:setup — extração do perfil

Primeira coisa a rodar. Cria a pasta Tino/ dentro do seu vault e sintetiza um perfil a partir do conteúdo do vault.

tino setup --vault "/Users/voce/Documents/MeuVault" --mock --force

O que acontece:

  1. Scanner heurístico percorre o vault recursivo e pontua arquivos por 6 sinais: nome (README/sobre/perfil/foco/CLAUDE/_INDEX), profundidade, recência, backlinks, tags no frontmatter, tamanho em palavras.
  2. Seleciona os top-N (default 30, ajuste com --top 50).
  3. Em modo mock aplica heurística determinística. Em modo real (dentro do Claude Code) o agent profile-extractor lê esses arquivos e sintetiza.
  4. Escreve rascunho em {vault}/Tino/_perfil.md com identidade, foco ativo, evita.

Abra {vault}/Tino/_perfil.md no Obsidian e revise. Esse arquivo é o coração do ranking — menos é mais, seja específico.

Exemplo de perfil bem calibrado. foco_ativo: [Claude Agent SDK, Managed Agents, Context engineering] é melhor que foco_ativo: [AI, machine learning, tools]. Termos específicos alimentam o matcher do ranker.

/tino:refresh — coletar e ranquear

tino refresh --vault "/Users/voce/Documents/MeuVault" --mock --force

Pipeline:

  1. Fetch das fontes em config/sources.default.yaml (20 feeds por default: Anthropic, OpenAI, Vercel, Supabase, HF, LangChain, Simon Willison, HN, Reddit, arXiv, …).
  2. Normaliza cada item em um schema comum {id, titulo, fonte, data, url, resumo_bruto}.
  3. Ranqueia item por item contra o perfil. Cada novidade vira um .md em {vault}/Tino/novidades/ com frontmatter estruturado + corpo com a justificativa ("por que importa para você").
  4. Incrementa counter em _config.md. Ao atingir 20 refreshes, sugere rodar /tino:profile-sync.

Flags úteis:

FlagEfeito
--mockRanker determinístico (heurística). Sem Claude. Bom pra testar.
--forceIgnora cache do dia e re-fetcha
--fixture-dir <path>Usa fixtures locais em vez de HTTP (offline/testes)
--limit <N>Máximo de items por fonte (default 20)
--since <ISO>Ignora items mais antigos que essa data

Abrindo o dashboard

tino dashboard:open --vault "/Users/voce/Documents/MeuVault"

Esse comando:

  1. Gera dashboard-data.json a partir do vault
  2. Sobe um HTTP server local em 127.0.0.1:8080 (se não tiver um rodando)
  3. Abre o browser

Três estados possíveis do dashboard:

  • Demo genérico — abriu direto sem ter rodado refresh em vault nenhum. Mostra 9 cards de exemplo hardcoded.
  • Demo com dados reais — o tino dashboard:open dumpou dashboard-data.json do vault. Dashboard renderiza as novidades reais já ranqueadas. Sem precisar conectar o vault.
  • Connected live — você clicou "Conectar vault" no topo e apontou pra pasta raiz do seu vault. Dashboard lê Tino/novidades/ ao vivo, e favoritar/thumbs-down reescrevem os .md.

Conectar o vault no dashboard

Passos:

  1. Clique em Conectar vault no topo direito.
  2. No seletor de pasta, escolha a raiz do seu vault Obsidian — não a subpasta Tino/. Ex: /Users/voce/Documents/MeuVault.
  3. Autorize leitura e escrita. O Tino só lê/escreve dentro de <vault>/Tino/.
  4. Se Tino/ não existir, o dashboard oferece criar a estrutura.
Browser lembra do vault. O handle é salvo em IndexedDB. Na próxima vez que abrir o dashboard, ele re-pede permissão (um clique) e reconecta automaticamente.
Dashboard conectado mas vazio? É porque a pasta <vault>/Tino/novidades/ está vazia. Rode tino refresh --vault <path> --mock --force no terminal. O empty state do dashboard mostra um diagnóstico detalhado do que foi encontrado.

Como o ranking funciona

Cada novidade recebe uma nota 0-10 e um veredito:

NotaVereditoSignificado
9.0–10.0FocaAltamente aplicável. Menciona item de foco_ativo ou resolve problema conhecido do vault.
7.0–8.9ConsideraRelevante mas não urgente.
5.0–6.9AcompanhaInteressante, fora do momento.
< 5.0IgnoreFora do contexto ou em evita.

Modo mock (default)

Heurística determinística em lib/rank-mock.mjs. Score base 5.0, ajustado por:

  • +3.0 por match em foco_ativo (cap +6)
  • +1.5 por match em identidade (cap +3)
  • -4.0 por match em evita
  • -1.5 por match com tokens de items com thumbs-down
  • +1.0 se publicado nos últimos 7 dias
  • Clip [0, 10], 1 casa decimal.

Modo real (Claude)

Dentro do Claude Code, o agent ranker usa o prompt canônico em config/prompts/rank-novelty.md pra ler perfil + item e produzir um JSON {nota, veredito, resumo, justificativa, cite}. A justificativa sempre cita um arquivo do vault que motivou a nota — auditável.

Customizando as fontes

20 fontes padrão vivem em config/sources.default.yaml (Anthropic News, OpenAI, DeepMind, Vercel, Supabase, HF, LangChain, Mistral, Simon Willison, HN, Reddit r/LocalLLaMA, r/ClaudeAI, arXiv cs.AI, Cohere, Latent Space, Import AI, Ben's Bites, The Rundown, HF Papers, GitHub trending).

Pra adicionar/remover, edite {vault}/Tino/_config.md:

---
frequencia: 1-2x por dia
---

## Fontes
- minha_fonte_custom    https://example.com/feed.xml
- outra_fonte           https://site.dev/rss

Na próxima refresh, o Tino dá merge do _config.md com o default. Fontes do vault ganham prioridade.

Feedback loop — afinando o ranking

Cada card tem três botões: Favoritar, Certeiro, Errou.

  • Favoritar marca favorito: true no .md da novidade. Desbloqueia /tino:deep-dive.
  • Certeiro grava em <vault>/Tino/_ajustes.md (seção ## Certeiros) e incrementa thumb_up.
  • Errou grava em ## Errados e incrementa thumb_down. Tokens do título entram no blacklist temporário do ranker.

Na próxima tino refresh, o ranker lê _ajustes.md e penaliza novidades novas que compartilhem tokens com os thumbed-down. Loop de treino honesto, local, inspecionável no próprio Obsidian.

Decisão arquitetural consciente. Thumbs-down não reescreve o _perfil.md automaticamente. O sinal fica em _ajustes.md — você decide se promove algo pra evita no perfil.

/tino:deep-dive — enriquecendo favoritos

Só funciona em items com favorito: true (controle de custo — deep-dive é caro em tokens se rodado em modo real).

tino deep-dive --vault "/Users/voce/Documents/MeuVault" --id "claude-agent-sdk-1-0" --mock

Escreve <vault>/Tino/favoritos/<id>-deep-dive.md com 3 seções:

  • Tutoriais de uso — passo-a-passo de como experimentar
  • Casos de sucesso — como outros usaram na prática
  • O que a comunidade está falando — sentimento + críticas

/tino:profile-sync — atualizando o perfil

Teu foco muda. Projetos entram e saem. A cada 20 refreshes o Tino sugere rodar:

tino profile-sync --vault "/Users/voce/Documents/MeuVault" --mock

Faz:

  1. Backup do _perfil.md atual em _perfil.backup-<timestamp>.md
  2. Re-escaneia o vault, regenera o perfil
  3. Preserva contadores (processadas, favoritadas, thumb_up/down, acerto)
  4. Reseta o counter de refreshes pra 0

Use --dry-run pra ver o diff antes de escrever.

FAQ & Troubleshooting

Conectei o vault no dashboard e ficou vazio

Significa que <vault>/Tino/novidades/ não tem .md. Rode tino refresh --vault <path> --mock --force. O empty state do dashboard mostra um diagnóstico detalhado.

Ranker tá dando "Acompanha" pra tudo

Perfil pouco calibrado. Abra <vault>/Tino/_perfil.md e certifique-se que foco_ativo, identidade e evita estão preenchidos com termos específicos. Depois rode tino refresh --force.

tino: command not found

Adicione ~/.local/bin ao PATH:

export PATH="$HOME/.local/bin:$PATH"

Ou rode bash install.sh de novo.

Erro "Firefox não suporta File System Access API"

Correto. Use Chrome, Edge, Arc ou Brave pro modo connected. Modo demo funciona em qualquer browser.

Posso apontar pra um vault qualquer que não seja Obsidian?

Sim. Qualquer pasta com .md serve. O scanner não depende do Obsidian.

Meu vault tem 5000+ notas e o scan tá lento

Use --top 30 (default) ou menos. O scanner ignora por default .obsidian/, Tino/, .git/, node_modules/.

Privacidade by design

  • Zero telemetria. Nada sai da tua máquina exceto os HTTP GET para os feeds RSS que tu mesmo configurou.
  • Zero conta, zero API key. Só precisa do Claude Code (que tu já tem).
  • Vault é read-only pro Tino fora da pasta Tino/. O Tino só escreve dentro da sua própria subpasta.
  • Dashboard não tem backend. HTML estático servido em loopback. Nenhuma requisição externa exceto CDN de fontes (Google Fonts).

Desinstalando

bash ~/tino-ai/uninstall.sh

Remove os symlinks (~/.claude/commands/tino-*.md, ~/.claude/agents/, ~/.local/bin/tino) e ~/.tino/config.sh. Repo e vault ficam intactos. Pra remover a pasta Tino/ do seu vault, delete manualmente — o Tino nunca faz isso sozinho.
